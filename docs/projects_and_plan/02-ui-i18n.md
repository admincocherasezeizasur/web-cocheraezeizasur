# 02 - UI, Maquetado e i18n Estatico

**Objetivo de esta fase:**
Construir la interfaz de usuario basandose en los diseños de Figma/PDF, implementar el enrutamiento de idiomas sin requerir Node.js y armar la navegacion principal.

**Instrucciones de Ejecucion:**

1. Estrategia i18n para SSG:
Dado que estamos usando Static Export, no se puede usar el middleware tradicional de Next.js para idiomas. 
Se debe crear una estructura de Dynamic Segments en el App Router: /app/[lang]/page.tsx.
Utilizar la funcion generateStaticParams para pre-renderizar los tres idiomas obligatorios: "es", "en", "pt".

2. Consumo de Diccionarios:
Crear una funcion utilitaria en /src/utils/getDictionary.ts que importe asincronamente el archivo JSON correspondiente desde /src/content/i18n/ segun el parametro [lang].

3. Desarrollo del Layout y Navbar:
Maquetar el Navbar superior. Debe incluir el logotipo de Cocheras Ezeiza Sur y un selector de idiomas (Banderas o texto ES/EN/PT).
El selector de idiomas debe funcionar como enlaces duros (tags anchor convencionales o Link de Next.js) que redirijan a la ruta /[lang] correspondiente, ya que no hay servidor para manejar el estado de la sesion.

4. Componentes Visuales Base:
Maquetar las secciones estaticas segun el diseño: 
- Hero Section (con la propuesta de valor "Tu auto seguro, tu viaje tranquilo").
- Seccion "Como Funciona" (Check-in rapido, Traslado, Regreso).
- Seccion de Testimonios.
- Footer con direccion (Venezuela 424, Ezeiza) y links de contacto.

5. Validacion:
Verificar que la navegacion entre /es, /en y /pt cambie los textos de la interfaz consumiendo los JSON, manteniendo la velocidad de carga instantanea.
